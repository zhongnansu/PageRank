<p><b>archive.is</b> (formerly <b>archive.today</b>) is an <a href="/wiki/Archive_site" title="Archive site">archive site</a> which stores snapshots of web pages.<sup class="reference" id="cite_ref-cnet_2-0"><a href="#cite_note-cnet-2">[2]</a></sup> It retrieves one page at a time similar to <a href="/wiki/WebCite" title="WebCite">WebCite</a>, smaller than 50 MB each, but with <a href="/wiki/Web_2.0" title="Web 2.0">Web 2.0</a> sites (such as <a href="/wiki/Google_Maps" title="Google Maps">Google Maps</a> and <a href="/wiki/Twitter" title="Twitter">Twitter</a>) included.
</p><p>Archive.is uses <a class="mw-redirect" href="/wiki/Headless_browsing" title="Headless browsing">headless browsing</a> to record what embedded resources need to be captured to provide a high-quality <a href="/wiki/Memento_Project" title="Memento Project">memento</a>, and creates a <a href="/wiki/Portable_Network_Graphics" title="Portable Network Graphics">PNG</a> image to provide a static and non-interactive visualization of the representation.<sup class="reference" id="cite_ref-3"><a href="#cite_note-3">[3]</a></sup>
</p><p>Unlike <a href="/wiki/Web_crawler" title="Web crawler">crawlers</a> such as <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a>, archive.is only captures individual pages in response to explicit user requests, and so does not obey the <a href="/wiki/Robots_exclusion_standard" title="Robots exclusion standard">robots exclusion standard</a>.<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> Because of this, website owners cannot unilaterally remove snapshots at will, making it a "permanent" archive.<sup class="reference" id="cite_ref-vice_stealing_5-0"><a href="#cite_note-vice_stealing-5">[5]</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag failed verification of its source citation(s). (November 2017)">not in citation given</span></a></i>]</sup>
</p><p>Since July 2013, archive.is supports the <a href="/wiki/Memento_Project" title="Memento Project">Memento Project</a> <a href="/wiki/Application_programming_interface" title="Application programming interface">application programming interface</a> (API).<sup class="reference" id="cite_ref-WSRG_6-0"><a href="#cite_note-WSRG-6">[6]</a></sup><sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup>
</p>

<h2><span class="mw-headline" id="Worldwide_availability">Worldwide availability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.is&amp;action=edit&amp;section=1" title="Edit section: Worldwide availability">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="China">China</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.is&amp;action=edit&amp;section=2" title="Edit section: China">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote navigation-not-searchable" role="note">See also: <a href="/wiki/Internet_censorship_in_China" title="Internet censorship in China">Internet censorship in China</a></div>
<p>According to <a href="/wiki/GreatFire" title="GreatFire">GreatFire.org</a>, archive.is has been blocked in China since March 2016,<sup class="reference" id="cite_ref-greatfire-archive-is_8-0"><a href="#cite_note-greatfire-archive-is-8">[8]</a></sup> archive.li since September 2017,<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> and archive.fo since July 2018.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>
</p>
<h3><span class="mw-headline" id="Finland">Finland</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.is&amp;action=edit&amp;section=3" title="Edit section: Finland">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote navigation-not-searchable" role="note">See also: <a class="mw-redirect" href="/wiki/Internet_censorship_in_Finland" title="Internet censorship in Finland">Internet censorship in Finland</a></div>
<p>On July 21, 2015, the operators blocked access to the service from all <a href="/wiki/Finland" title="Finland">Finnish</a> <a href="/wiki/IP_address" title="IP address">IP addresses</a>, stating on <a href="/wiki/Twitter" title="Twitter">Twitter</a> that they did this in order to avoid escalating a dispute they allegedly had with the Finnish government.<sup class="reference" id="cite_ref-ilta_Suom_11-0"><a href="#cite_note-ilta_Suom-11">[11]</a></sup>
</p>
<h3><span class="mw-headline" id="Russia">Russia</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.is&amp;action=edit&amp;section=4" title="Edit section: Russia">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="hatnote navigation-not-searchable" role="note">See also: <a href="/wiki/Internet_censorship_in_Russia" title="Internet censorship in Russia">Internet censorship in Russia</a></div>
<p>In <a href="/wiki/Russia" title="Russia">Russia</a>, only HTTP access is possible; HTTPS connections are blocked.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup><sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup>
</p>
<h2><span class="mw-headline" id="Hosting">Hosting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.is&amp;action=edit&amp;section=5" title="Edit section: Hosting">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Veesp currently hosts archive.is.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup> The site was protected by Cloudflare from August 16 2018 until September 2 2018.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2018)">citation needed</span></a></i>]</sup>
</p>
<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.is&amp;action=edit&amp;section=6" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Archive.is records only text and images, excluding video and other <a href="/wiki/Static_web_page" title="Static web page">non-static</a> content. It keeps track of the history of snapshots saved, returning to the user a request for confirmation before adding a new snapshot of an already saved Internet address.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup>
</p><p>Web pages can't be <a href="/wiki/Redundancy_(engineering)" title="Redundancy (engineering)">duplicated</a> from <i>http://archive.is</i> to <i>http://www.archive.org</i> as <a href="/wiki/Remote_backup_service" title="Remote backup service">second-level backup</a>. The reverse - from <i>www.archive.org</i> to <i>archive.is</i> - is possible,<sup class="reference" id="cite_ref-Wayback_Mach_to_archive.is_16-0"><a href="#cite_note-Wayback_Mach_to_archive.is-16">[16]</a></sup> but the copy usually takes more time than a direct capture. Some web sites can't be saved by either <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a> or archive.fo due to their <a class="mw-redirect" href="/wiki/Robots.txt" title="Robots.txt">robots.txt</a> file.
</p><p>The research toolbar enables advanced keywords operators, using  
<code class="mw-highlight" dir="ltr" id="" style="">*</code> as the <a href="/wiki/Wildcard_character#Computing" title="Wildcard character">wildcard character</a>. 
A couple of <a href="/wiki/Quotation_mark" title="Quotation mark">quotation marks</a> address the search to an exact sequence of keywords present in the title or in the body of the webpage, whereas the <i>insite</i> operator restricts it to a specific Internet domain<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup>.
</p>

<p><b>DMOZ</b> (from <i>directory.mozilla.org</i>, an earlier <a href="/wiki/Domain_name" title="Domain name">domain name</a>) was a multilingual <a href="/wiki/Open_content" title="Open content">open-content</a> <a href="/wiki/Web_directory" title="Web directory">directory</a> of <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> links. The site and community who maintained it were also known as the <b>Open Directory Project</b> (<b>ODP</b>). It was owned by <a href="/wiki/AOL" title="AOL">AOL</a> (now a part of <a class="mw-redirect" href="/wiki/Verizon" title="Verizon">Verizon</a>'s <a href="/wiki/Oath_Inc." title="Oath Inc.">Oath Inc.</a>) but constructed and maintained by a <a href="/wiki/Virtual_community" title="Virtual community">community</a> of volunteer editors.
</p><p>DMOZ used a hierarchical <a href="/wiki/Ontology_(information_science)" title="Ontology (information science)">ontology</a> scheme for organizing site listings. Listings on a similar topic were grouped into categories which then included smaller categories.
</p><p>DMOZ closed on March 17, 2017 because AOL no longer wished to support the project.<sup class="reference" id="cite_ref-2"><a href="#cite_note-2">[2]</a></sup><sup class="reference" id="cite_ref-resource-zone_3-0"><a href="#cite_note-resource-zone-3">[3]</a></sup>  The website became a single landing page on that day, with links to a static archive of DMOZ, and to the DMOZ discussion forum, where plans to rebrand and relaunch the directory are being discussed.<sup class="reference" id="cite_ref-resource-zone_3-1"><a href="#cite_note-resource-zone-3">[3]</a></sup>
</p><p>As of  September 2017<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=DMOZ&amp;action=edit">[update]</a></sup>, a non-editable mirror remained available at dmoztools.net,<sup class="reference" id="cite_ref-4"><a href="#cite_note-4">[4]</a></sup> and it was stated that while the DMOZ URL would not return, a successor version of the directory would, at Curlie.org.<sup class="reference" id="cite_ref-5"><a href="#cite_note-5">[5]</a></sup><sup class="reference" id="cite_ref-6"><a href="#cite_note-6">[6]</a></sup>
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>DMOZ was founded in the United States as <b>Gnuhoo</b> by <a href="/wiki/Rich_Skrenta" title="Rich Skrenta">Rich Skrenta</a> and <a href="/wiki/Bob_Truel" title="Bob Truel">Bob Truel</a> in 1998 while they were both working as engineers for <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a>. Chris Tolles, who worked at Sun Microsystems as the head of marketing for network security products, also signed on in 1998 as a co-founder of Gnuhoo along with co-founders Bryn Dole and Jeremy Wenokur. Skrenta had developed <a class="new" href="/w/index.php?title=TASS_(software)&amp;action=edit&amp;redlink=1" title="TASS (software) (page does not exist)">TASS</a>, an ancestor of <a href="/wiki/Tin_(newsreader)" title="Tin (newsreader)">tin</a>, the popular threaded <a href="/wiki/Usenet" title="Usenet">Usenet</a> newsreader for <a href="/wiki/Unix" title="Unix">Unix</a> systems. The original category structure of the Gnuhoo directory was based loosely on the structure of Usenet newsgroups then in existence.
</p><p>The Gnuhoo directory went live on June 5, 1998. After Richard Stallman and the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> objected to the use of "Gnu" in the name, Gnuhoo was changed to NewHoo.<sup class="reference" id="cite_ref-SlashdotGnuhoo_7-0"><a href="#cite_note-SlashdotGnuhoo-7">[7]</a></sup> <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a> then objected to the use of "Hoo" in the name, prompting a proposed name change, to ZURL.<sup class="reference" id="cite_ref-8"><a href="#cite_note-8">[8]</a></sup> Prior to switching to ZURL, NewHoo was acquired by <a href="/wiki/Netscape" title="Netscape">Netscape Communications Corporation</a> in October 1998 and became the Open Directory Project. Netscape released Open Directory data under the <a href="/wiki/Open_Directory_License" title="Open Directory License">Open Directory License</a>. Netscape was acquired by <a href="/wiki/AOL" title="AOL">AOL</a> shortly thereafter and DMOZ was one of the assets included in the acquisition.
</p>
<div class="thumb tright"><div class="thumbinner" style="width:402px;"><a class="image" href="/wiki/File:Odp_sitecount_top.png"><img alt="" class="thumbimage" data-file-height="366" data-file-width="801" height="183" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/1e/Odp_sitecount_top.png/400px-Odp_sitecount_top.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/1e/Odp_sitecount_top.png/600px-Odp_sitecount_top.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/1e/Odp_sitecount_top.png/800px-Odp_sitecount_top.png 2x" width="400"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Odp_sitecount_top.png" title="Enlarge"></a></div>DMOZ size by date, 1998 to 2015.</div></div></div>
<p>By the time Netscape assumed stewardship, the Open Directory Project had about 100,000 <a class="mw-redirect" href="/wiki/Uniform_Resource_Locator" title="Uniform Resource Locator">URLs</a> indexed with contributions from about 4500 editors. On October 5, 1999, the number of URLs indexed by DMOZ reached one million. According to an unofficial estimate, the URLs in DMOZ numbered 1.6 million in April 2000, surpassing those in the <a href="/wiki/Yahoo!_Directory" title="Yahoo! Directory">Yahoo! Directory</a>.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup> DMOZ achieved the milestones of indexing two million URLs on August 14, 2000, three million listings on November 18, 2001 and four million on December 3, 2003. As of April 2013 there were 5,169,995 sites listed in over 1,017,500 categories. On October 31, 2015, there were 3,996,412 sites listed in 1,026,706 categories.
</p><p>In January 2006, DMOZ began publishing online reports to inform the public about the development of the project. The first report covered the year 2005. Monthly reports were issued subsequently until September 2006.<sup class="reference" id="cite_ref-odpreports_10-0"><a href="#cite_note-odpreports-10">[10]</a></sup> These reports gave greater insight into the functioning of the directory than the simplified statistics provided on the front page of the directory. The number of listings and categories cited on the front page included "Test" and "Bookmarks" categories but these were not included in the RDF dump offered to users. There were about 7330 active editors during August 2006.<sup class="reference" id="cite_ref-odpreports_10-1"><a href="#cite_note-odpreports-10">[10]</a></sup> 75,151 editors had contributed to the directory as of March 31, 2007.<sup class="reference" id="cite_ref-frontpage_11-0"><a href="#cite_note-frontpage-11">[11]</a></sup> As of April 2013, the number of contributing editors had increased to 97,584.<sup class="reference" id="cite_ref-frontpage_11-1"><a href="#cite_note-frontpage-11">[11]</a></sup>
</p>
<h3><span id="System_failure_and_editing_outage.2C_October_to_December_2006"></span><span class="mw-headline" id="System_failure_and_editing_outage,_October_to_December_2006">System failure and editing outage, October to December 2006</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=2" title="Edit section: System failure and editing outage, October to December 2006">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>On October 20, 2006, DMOZ's main server suffered a <a href="/wiki/Catastrophic_failure" title="Catastrophic failure">catastrophic failure</a><sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> that prevented editors from working on the directory until December 18, 2006. During that period, an older build of the directory was visible to the public. On January 13, 2007, the Site Suggestion and Update Listings forms were again made available.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup> On January 26, 2007, weekly publication of RDF dumps resumed. To avoid future outages, the system resided on a redundant configuration of two Intel-based servers from then on.<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup>
</p><p>The site's interface was given an upgrade in 2016, branded "DMOZ 3.0" but AOL took it offline the following year.
</p>
<h3><span class="mw-headline" id="Competing_and_spinoff_projects">Competing and spinoff projects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=3" title="Edit section: Competing and spinoff projects">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>As DMOZ became more widely known, two other major <a href="/wiki/Web_directory" title="Web directory">web directories</a> edited by volunteers and sponsored by <a href="/wiki/Go.com" title="Go.com">Go.com</a> and <a href="/wiki/Zeal_(web)" title="Zeal (web)">Zeal</a> emerged, both now defunct. These directories did not license their content for <a href="/wiki/Open_content" title="Open content">open content</a> distribution.<sup class="reference" id="cite_ref-15"><a href="#cite_note-15">[15]</a></sup><sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup>
</p><p>The concept of using a large-scale community of editors to compile online content has been successfully applied to other types of projects. DMOZ's editing model directly inspired at least three other open content volunteer projects: music site MusicMoz, an open content restaurant directory known as <a href="/wiki/ChefMoz" title="ChefMoz">ChefMoz</a><sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup> and an encyclopedia known as <a href="/wiki/Open_Site" title="Open Site">Open Site</a>.<sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup> Finally, according to <a href="/wiki/Larry_Sanger" title="Larry Sanger">Larry Sanger</a>, DMOZ was part of the inspiration for the <a href="/wiki/Nupedia" title="Nupedia">Nupedia</a> project, out of which <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a> grew.<sup class="reference" id="cite_ref-19"><a href="#cite_note-19">[19]</a></sup>
</p>

<h2><span class="mw-headline" id="Content">Content</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=5" title="Edit section: Content">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Gnuhoo borrowed the basic outline for its initial ontology from <a href="/wiki/Usenet" title="Usenet">Usenet</a>. In 1998, Rich Skrenta said, "I took a long list of groups and hand-edited them into a hierarchy."<sup class="reference" id="cite_ref-20"><a href="#cite_note-20">[20]</a></sup> For example, the topic covered by the comp.ai.alife newsgroup was represented by the category Computers/AI/Artificial_Life. The original divisions were for <i>Adult</i>, <i>Arts</i>, <i>Business</i>, <i>Computers</i>, <i>Games</i>, <i>Health</i>, <i>Home</i>, <i>News</i>, <i>Recreation</i>, <i>Reference</i>, <i>Regional</i>, <i>Science</i>, <i>Shopping</i>, <i>Society</i>, <i>Sports</i> and "World". While these sixteen <i>top-level</i> categories have remained intact, the ontology of second- and lower-level categories has undergone a gradual evolution; significant changes are initiated by discussion among editors and then implemented when consensus has been reached.
</p><p>In July 1998, the directory became <a href="/wiki/Multilingualism" title="Multilingualism">multilingual</a> with the addition of the <i>World</i> top-level category. The remainder of the directory lists only English language sites. By May 2005, seventy-five languages were represented. The growth rate of the non-English components of the directory has been greater than the English component since 2002. While the English component of the directory held almost 75% of the sites in 2003, the <i>World</i> level grew to over 1.5 million sites as of May 2005, forming roughly one-third of the directory. The ontology in non-English categories generally mirrors that of the English directory, although exceptions which reflect language differences are quite common.
</p><p>Several of the top-level categories have unique characteristics. The <i>Adult</i> category is not present on the directory homepage but it is fully available in the RDF dump that DMOZ provides. While the bulk of the directory is categorized primarily by topic, the Regional category is categorized primarily by region. This has led many to view DMOZ as two parallel directories: <i>Regional</i> and <i>Topical</i>.
</p><p>On November 14, 2000, a special directory within DMOZ was created for people under 18 years of age.<sup class="reference" id="cite_ref-21"><a href="#cite_note-21">[21]</a></sup> Key factors distinguishing this "Kids and Teens" area from the main directory are:
</p>
<ul><li>stricter guidelines which limit the listing of sites to those which are targeted or "appropriate" for people under 18 years of age;<sup class="reference" id="cite_ref-22"><a href="#cite_note-22">[22]</a></sup></li>
<li>category names as well as site descriptions use vocabulary which is "<a class="mw-redirect" href="/wiki/Age_appropriate" title="Age appropriate">age appropriate</a>";</li>
<li><i>age tags</i> on each listing distinguish content appropriate for kids (age 12 and under), teens (13 to 15 years old) and mature teens (16 to 18 years old);</li>
<li>Kids and Teens content is available as a separate RDF dump;</li>
<li>editing permissions are such that the community is parallel to that of DMOZ.</li></ul>
<p>By May 2005, this portion of DMOZ included over 32,000 site listings.
</p><p>Since early 2004, the whole site has been in <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> encoding. Prior to this, the encoding used to be <a class="mw-redirect" href="/wiki/ISO_8859-1" title="ISO 8859-1">ISO 8859-1</a> for English language categories and a language-dependent character set for other languages. The RDF dumps have been encoded in UTF-8 since early 2000.
</p>
<h3><span class="mw-headline" id="Maintenance">Maintenance</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=6" title="Edit section: Maintenance">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Directory listings are maintained by editors. While some editors focus on the addition of new listings, others focus on maintaining the existing listings and some do both. This includes tasks such as the editing of individual listings to correct spelling and/or grammatical errors, as well as monitoring the status of linked sites. Still others go through site submissions to remove spam and duplicate submissions.
</p><p>Robozilla is a <a href="/wiki/Web_crawler" title="Web crawler">Web crawler</a> written to check the status of all sites listed in DMOZ. Periodically, Robozilla will flag sites which appear to have moved or disappeared and editors follow up to check the sites and take action. This process is critical for the directory in striving to achieve one of its founding goals: to reduce the <a href="/wiki/Link_rot" title="Link rot">link rot</a> in web directories. Shortly after each run, the sites marked with errors are automatically moved to the unreviewed queue where editors may investigate them when time permits.
</p><p>Due to the popularity of DMOZ and its resulting impact on <a href="/wiki/Web_search_engine" title="Web search engine">search engine</a> rankings (See <a href="/wiki/PageRank" title="PageRank">PageRank</a>), domains with lapsed registration that are listed on DMOZ have attracted <a href="/wiki/Domain_hijacking" title="Domain hijacking">domain hijacking</a>, an issue that has been addressed by regularly removing expired domains from the directory.
</p><p>While corporate funding and staff for DMOZ have diminished in recent years, volunteers have created editing tools such as linkcheckers to supplement Robozilla, category crawlers, spellcheckers, search tools that directly sift a recent RDF dump, <a href="/wiki/Bookmarklet" title="Bookmarklet">bookmarklets</a> to help automate some editing functions, mozilla based add-ons,<sup class="reference" id="cite_ref-23"><a href="#cite_note-23">[23]</a></sup> and tools to help work through unreviewed queues.
</p>
<h3><span class="mw-headline" id="License_and_requirements">License and requirements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=7" title="Edit section: License and requirements">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>DMOZ data was previously made available under the terms of the <a href="/wiki/Open_Directory_License" title="Open Directory License">Open Directory License</a>, which required a specific DMOZ attribution table on every Web page that uses the data.
</p><p>The Open Directory License also included a requirement that users of the data continually check DMOZ site for updates and discontinue use and distribution of the data or works derived from the data once an update occurs. This restriction prompted the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> to refer to the <a href="/wiki/Open_Directory_License" title="Open Directory License">Open Directory License</a> as a non-free documentation license, citing the right to redistribute a given version not being permanent and the requirement to check for changes to the license.
</p><p>In 2011, DMOZ silently changed its license to a <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a> Attribution license<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup>, which is a free license (and GPL compatible).<sup class="reference" id="cite_ref-24"><a href="#cite_note-24">[24]</a></sup>
</p>
<h3><span class="mw-headline" id="RDF_dumps">RDF dumps</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=8" title="Edit section: RDF dumps">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>DMOZ data is made available through an RDF-like dump that is published on a download server, older versions are also archived there.<sup class="reference" id="cite_ref-25"><a href="#cite_note-25">[25]</a></sup> New versions are usually generated weekly. An DMOZ editor has catalogued a number of bugs that are encountered in the DMOZ RDF dump, most importantly that the file format isn't RDF.<sup class="reference" id="cite_ref-26"><a href="#cite_note-26">[26]</a></sup> So while today the so-called RDF dump is valid <a href="/wiki/XML" title="XML">XML</a>, it is not valid <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">RDF</a> and as such, software to process the DMOZ RDF dump needs to be specifically written for DMOZ data.
</p>
<h3><span class="mw-headline" id="Content_users">Content users</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=9" title="Edit section: Content users">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>DMOZ data powers the core <a href="/wiki/Directory_service" title="Directory service">directory services</a> for many of the Web's largest search engines and portals, including Netscape Search, AOL Search, and <a href="/wiki/Alexa_Internet" title="Alexa Internet">Alexa</a>. <a href="/wiki/Google_Directory" title="Google Directory">Google Directory</a> used DMOZ information, until being shuttered in July 2011.<sup class="reference" id="cite_ref-27"><a href="#cite_note-27">[27]</a></sup>
</p><p>Other uses are also made of DMOZ data. For example, in the spring of 2004 <a href="/wiki/Yahoo!_Search_Marketing" title="Yahoo! Search Marketing">Overture</a> announced a search service for third parties combining Yahoo! Directory search results with DMOZ titles, descriptions and category metadata. The search engine Gigablast announced on May 12, 2005 its searchable copy of DMOZ. The technology permits search of websites listed in specific categories, "in effect, instantly creating over 500,000 vertical search engines".<sup class="reference" id="cite_ref-28"><a href="#cite_note-28">[28]</a></sup>
</p><p>As of  8 September 2006<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=DMOZ&amp;action=edit">[update]</a></sup>, DMOZ listed 313 English-language Web sites that use DMOZ data as well as 238 sites in other languages.<sup class="reference" id="cite_ref-29"><a href="#cite_note-29">[29]</a></sup> However, these figures do not reflect the full picture of use, as those sites that use DMOZ data without following the terms of the DMOZ license are not listed.
</p>
<h2><span class="mw-headline" id="Policies_and_procedures">Policies and procedures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=10" title="Edit section: Policies and procedures">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright"><div class="thumbinner" style="width:187px;"><a class="image" href="/wiki/File:Rich_Skrenta.jpg"><img alt="" class="thumbimage" data-file-height="180" data-file-width="185" height="180" src="//upload.wikimedia.org/wikipedia/commons/4/41/Rich_Skrenta.jpg" width="185"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:Rich_Skrenta.jpg" title="Enlarge"></a></div>DMOZ was co-founded by <a href="/wiki/Rich_Skrenta" title="Rich Skrenta">Rich Skrenta</a> (depicted in 2009, age 42).</div></div></div>
<p>Restrictions are imposed on who can become an DMOZ editor. The primary gatekeeping mechanism is an editor application process wherein editor candidates demonstrate their editing abilities, disclose affiliations that might pose a <a href="/wiki/Conflict_of_interest" title="Conflict of interest">conflict of interest</a>, and otherwise give a sense of how the applicant would likely mesh with the DMOZ culture and mission.<sup class="reference" id="cite_ref-30"><a href="#cite_note-30">[30]</a></sup> A majority of applications are rejected but reapplying is allowed and sometimes encouraged. The same standards apply to editors of all categories and subcategories.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2016)">citation needed</span></a></i>]</sup>
</p><p>DMOZ's editing model is a <a class="mw-redirect" href="/wiki/Hierarchal_structure" title="Hierarchal structure">hierarchical</a> one. Upon becoming editors, individuals will generally have editing permissions in only a small category. Once they have demonstrated basic editing skills in compliance with the Editing Guidelines, they are welcome to apply for additional editing privileges in either a broader category or else another category in the directory. Mentorship relationships between editors are encouraged, and internal forums provide a vehicle for new editors to ask questions.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2016)">citation needed</span></a></i>]</sup>
</p><p>DMOZ has its own internal forums, the contents of which are intended only for editors to communicate with each other primarily about editing topics. Access to the forums requires an editor account and editors are expected to keep the contents of these forums private.<sup class="reference" id="cite_ref-commguide_31-0"><a href="#cite_note-commguide-31">[31]</a></sup>
</p><p>Over time, senior editors can be granted additional privileges which reflect their editing experience and leadership within the editing community. The most straightforward are <i>editall</i> privileges, which allow an editor to access all categories in the directory. <i>Meta</i> privileges additionally allow editors to perform tasks such as reviewing editor applications, setting category features, and handling external and internal abuse reports. <i>Cateditall</i> privileges are similar to <i>editall</i>, but only for a single directory category. Similarly, <i>catmod</i> privileges are similar to <i>meta</i>, but only for a single directory category. <i>Catmv</i> privileges allow editors to make changes to directory ontology by moving or renaming categories. All of these privileges are granted by admins and staff, usually after discussion with <i>meta</i> editors.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2016)">citation needed</span></a></i>]</sup>
</p><p>In August 2004, a new level of privileges called <i>admin</i> was introduced. Administrator status was granted to a number of long serving metas by staff. Administrators have the ability to grant editall+ privileges to other editors and to approve new directory-wide policies, powers which had previously only been available to root (staff) editors.<sup class="reference" id="cite_ref-adminguide_32-0"><a href="#cite_note-adminguide-32">[32]</a></sup>
</p><p>All DMOZ editors are expected to abide by DMOZ's Editing Guidelines. These guidelines describe editing basics: which types of sites may be listed and which may not; how site listings should be titled and described in a loosely consistent manner; conventions for the naming and building of categories; conflict of interest limitations on the editing of sites which the editor may own or otherwise be affiliated with; and a code of conduct within the community.<sup class="reference" id="cite_ref-33"><a href="#cite_note-33">[33]</a></sup> Editors who are found to have violated these guidelines may be contacted by staff or senior editors, have their editing permissions cut back, or lose their editing privileges entirely. DMOZ Guidelines are periodically revised after discussion in editor forums.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2016)">citation needed</span></a></i>]</sup>
</p>
<h2><span class="mw-headline" id="Controversy_and_criticism">Controversy and criticism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=11" title="Edit section: Controversy and criticism">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There have long been allegations that volunteer DMOZ editors give favorable treatment to their own websites while concomitantly thwarting the good faith efforts of their competition.<sup class="reference" id="cite_ref-34"><a href="#cite_note-34">[34]</a></sup> Such allegations are fielded by ODP's staff and meta editors, who have the authority to take disciplinary action against volunteer editors who are suspected of engaging in abusive editing practices.<sup class="reference" id="cite_ref-35"><a href="#cite_note-35">[35]</a></sup> In 2003, DMOZ introduced a new <i>Public Abuse Report System</i> that allows members of the general public to report and track allegations of abusive editor conduct using an online form.<sup class="reference" id="cite_ref-36"><a href="#cite_note-36">[36]</a></sup> Uninhibited discussion of DMOZ's purported shortcomings has become more common on mainstream <a href="/wiki/Webmaster" title="Webmaster">webmaster</a> discussion forums. Although site policies suggest that an individual site should be submitted to only one category,<sup class="reference" id="cite_ref-37"><a href="#cite_note-37">[37]</a></sup> as of October 2007, Topix.com, a news aggregation site operated by DMOZ founder Rich Skrenta, had more than 17,000 listings.<sup class="reference" id="cite_ref-38"><a href="#cite_note-38">[38]</a></sup>
</p><p>Early in the history of DMOZ, its staff gave representatives of selected companies, such as <i><a href="/wiki/Rolling_Stone" title="Rolling Stone">Rolling Stone</a></i> or <a href="/wiki/CNN" title="CNN">CNN</a>, editing access in order to list individual pages from their websites.<sup class="reference" id="cite_ref-39"><a href="#cite_note-39">[39]</a></sup> Links to individual CNN articles were added until 2004, but were entirely removed from the directory in January 2008<sup class="reference" id="cite_ref-40"><a href="#cite_note-40">[40]</a></sup> due to the content being outdated and not considered worth the effort to maintain. There have been no similar experiments with the editing policy since then.
</p>
<h3><span class="mw-headline" id="Ownership_and_management">Ownership and management</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=12" title="Edit section: Ownership and management">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<div class="thumb tright"><div class="thumbinner" style="width:292px;"><a class="image" href="/wiki/File:DMOZ_Closed.png"><img alt="" class="thumbimage" data-file-height="593" data-file-width="771" height="223" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/27/DMOZ_Closed.png/290px-DMOZ_Closed.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/27/DMOZ_Closed.png/435px-DMOZ_Closed.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/27/DMOZ_Closed.png/580px-DMOZ_Closed.png 2x" width="290"/></a> <div class="thumbcaption"><div class="magnify"><a class="internal" href="/wiki/File:DMOZ_Closed.png" title="Enlarge"></a></div>Screenshot taken in April 2017 showing DMOZ website to be closed.</div></div></div>
<p>Underlying some controversy surrounding DMOZ is its ownership and management. Some of the original GnuHoo volunteers felt that they had been deceived into joining a commercial enterprise.<sup class="reference" id="cite_ref-SlashdotGnuhoo_7-1"><a href="#cite_note-SlashdotGnuhoo-7">[7]</a></sup> To varying degrees, those complaints have continued up until the present.
</p><p>At DMOZ's inception, there was little thought given to the idea of how DMOZ should be managed and there were no official <a href="/wiki/Internet_forum" title="Internet forum">forums</a>, guidelines or <a href="/wiki/FAQ" title="FAQ">FAQs</a>. In essence, DMOZ began as a free for all.<sup class="reference" id="cite_ref-41"><a href="#cite_note-41">[41]</a></sup>
</p><p>As time went on, the ODP Editor Forums became the <i>de facto</i> DMOZ parliament and when one of DMOZ's staff members would post an opinion in the forums, it would be considered an official ruling.<sup class="reference" id="cite_ref-commguide_31-1"><a href="#cite_note-commguide-31">[31]</a></sup> Even so, DMOZ staff began to give trusted senior editors additional editing privileges, including the ability to approve new editor applications, which eventually led to a stratified hierarchy of duties and privileges among DMOZ editors, with DMOZ's paid staff having the final say regarding DMOZ's policies and procedures.<sup class="reference" id="cite_ref-adminguide_32-1"><a href="#cite_note-adminguide-32">[32]</a></sup><sup class="reference" id="cite_ref-metaguide_42-0"><a href="#cite_note-metaguide-42">[42]</a></sup>
</p><p>Robert Keating, a principal of Touchstone Consulting Group in Washington, D.C. since 2006, has worked as AOL's Program Manager for DMOZ since 2004. He started working for AOL in 1999 as Senior Editor for AOL Search, then as Managing Editor, AOL Search, DMOZ, and then as Media Ecosystem Manager, AOL Product Marketing.<sup class="reference" id="cite_ref-43"><a href="#cite_note-43">[43]</a></sup><sup class="reference" id="cite_ref-44"><a href="#cite_note-44">[44]</a></sup>
</p>
<h3><span class="mw-headline" id="Editor_removal_procedures">Editor removal procedures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=13" title="Edit section: Editor removal procedures">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>DMOZ's editor removal procedures are overseen by DMOZ's staff and meta editors. According to DMOZ's official editorial guidelines, editors are removed for abusive editing practices or uncivil behaviour. Discussions that may result in disciplinary action against volunteer editors take place in a private forum which can only be accessed by DMOZ's staff and meta editors. Volunteer editors who are being discussed are not given notice that such proceedings are taking place.<sup class="reference" id="cite_ref-metaguide_42-1"><a href="#cite_note-metaguide-42">[42]</a></sup> Some people find this arrangement distasteful, wanting instead a discussion modeled more like a trial held in the U.S. judicial system.<sup class="reference" id="cite_ref-yahooxodp_45-0"><a href="#cite_note-yahooxodp-45">[45]</a></sup>
</p><p>In the article "Editor Removal Explained", DMOZ meta editor Arlarson states that "a great deal of confusion about the removal of editors from DMOZ results from false or misleading statements by former editors".<sup class="reference" id="cite_ref-46"><a href="#cite_note-46">[46]</a></sup>
</p><p>The DMOZ's confidentiality guidelines prohibit any current DMOZ editors in a position to know anything from discussing the reasons for specific editor removals.<sup class="reference" id="cite_ref-metaguide_42-2"><a href="#cite_note-metaguide-42">[42]</a></sup> However, a generic list of reasons is for example given in the guidelines.<sup class="reference" id="cite_ref-47"><a href="#cite_note-47">[47]</a></sup> In the past, this has led to removed DMOZ editors wondering why they cannot login at DMOZ to perform their editing work.<sup class="reference" id="cite_ref-48"><a href="#cite_note-48">[48]</a></sup><sup class="reference" id="cite_ref-49"><a href="#cite_note-49">[49]</a></sup>
</p>
<h3><span class="mw-headline" id="Allegations_that_editors_are_removed_for_criticizing_policies">Allegations that editors are removed for criticizing policies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=14" title="Edit section: Allegations that editors are removed for criticizing policies">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>David F. Prenatt, Jr., former DMOZ editor <i>netesq</i>, and another former editor known by the alias <i>The Cunctator</i>, both claim to have been removed for disagreeing with staff about changes to policies, particularly DMOZ's <a href="/wiki/Copyright" title="Copyright">copyright</a> policies. According to their claims, staff use the excuse of uncivil behaviour as a means to remove bothersome editors.<sup class="reference" id="cite_ref-yahooxodp_45-1"><a href="#cite_note-yahooxodp-45">[45]</a></sup><sup class="reference" id="cite_ref-50"><a href="#cite_note-50">[50]</a></sup><sup class="reference" id="cite_ref-51"><a href="#cite_note-51">[51]</a></sup>
</p>
<h3><span class="mw-headline" id="Blacklisting_allegations">Blacklisting allegations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=15" title="Edit section: Blacklisting allegations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Senior DMOZ editors have the ability to attach "warning" or "do not list" notes to individual domains but no editor has the unilateral ability to block certain sites from being listed. Sites with these notes might still be listed and at times notes are removed after some discussion.<sup class="reference" id="cite_ref-52"><a href="#cite_note-52">[52]</a></sup>
</p>
<h3><span class="mw-headline" id="Hierarchical_structure">Hierarchical structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=16" title="Edit section: Hierarchical structure">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Criticism of DMOZ's <a class="mw-redirect" href="/wiki/Hierarchical" title="Hierarchical">hierarchical</a> structure emerged by around 2005. Many believe hierarchical directories are too complicated. With the emergence of <a href="/wiki/Web_2.0" title="Web 2.0">Web 2.0</a>, <a href="/wiki/Folksonomy" title="Folksonomy">folksonomies</a> began to appear, and some editors proposed that folksonomies, <a class="mw-redirect" href="/wiki/Network_(mathematics)" title="Network (mathematics)">networks</a> and <a href="/wiki/Directed_graph" title="Directed graph">directed graphs</a> are more "natural" and easier to manage than hierarchies.<sup class="reference" id="cite_ref-53"><a href="#cite_note-53">[53]</a></sup><sup class="reference" id="cite_ref-54"><a href="#cite_note-54">[54]</a></sup><sup class="reference" id="cite_ref-55"><a href="#cite_note-55">[55]</a></sup>
</p>
<h2><span class="mw-headline" id="Software">Software</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=17" title="Edit section: Software">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<h3><span class="mw-headline" id="Search">Search</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=18" title="Edit section: Search">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The ODPSearch software is a derivative version of <a href="/wiki/Isearch" title="Isearch">Isearch</a> which is <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a>, licensed under the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">Mozilla Public License</a>.<sup class="reference" id="cite_ref-56"><a href="#cite_note-56">[56]</a></sup>
</p>
<h3><span class="mw-headline" id="Editor_forums">Editor forums</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=19" title="Edit section: Editor forums">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The ODP Editor Forums were originally run on software that was based on the proprietary <a class="new" href="/w/index.php?title=Ultimate_Bulletin_Board&amp;action=edit&amp;redlink=1" title="Ultimate Bulletin Board (page does not exist)">Ultimate Bulletin Board</a> system. In June 2003, they switched to the open source <a href="/wiki/PhpBB" title="PhpBB">phpBB</a> system. As of 2007, these forums are powered by a modified version of phpBB.
</p>
<h3><span class="mw-headline" id="Bug_tracking">Bug tracking</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=20" title="Edit section: Bug tracking">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The bug tracking software used by the ODP is <a href="/wiki/Bugzilla" title="Bugzilla">Bugzilla</a> and the web server <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache</a>. <a class="mw-redirect" href="/wiki/Squid_cache" title="Squid cache">Squid</a> web proxy server was also used but it was removed in August 2007 when the storage servers were reorganized. All these applications are open source.
</p>
<h3><span class="mw-headline" id="Interface">Interface</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=DMOZ&amp;action=edit&amp;section=21" title="Edit section: Interface">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>The DMOZ database/editing software is closed source (although Richard Skrenta has said in June 1998 that he was considering licensing it under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a>). This has led to criticism from the aforementioned GNU project, many of whom also criticized the DMOZ content license.  The content was later released under a Creative Commons license, which is compatible with the GNU license.<sup class="reference" id="cite_ref-57"><a href="#cite_note-57">[57]</a></sup>
</p><p>As such, there have been some efforts to provide alternatives to DMOZ. These alternatives would allow communities of like-minded editors to set up and maintain their own open source/open content Web directories. However, no significant open source/open content alternative to DMOZ has emerged.
</p>

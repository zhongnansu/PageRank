<p><b>Evergreen</b> is an <a class="mw-redirect" href="/wiki/Open_source" title="Open source">open source</a> <a href="/wiki/Integrated_library_system" title="Integrated library system">Integrated Library System</a> (ILS), initially developed by the <a href="/wiki/Georgia_Public_Library_Service" title="Georgia Public Library Service">Georgia Public Library Service</a> for <a href="/wiki/Public_Information_Network_for_Electronic_Services" title="Public Information Network for Electronic Services">Public Information Network for Electronic Services</a> (PINES), a statewide resource-sharing consortium with over 270 member libraries.<sup class="reference" id="cite_ref-libraryjournal_1-0"><a href="#cite_note-libraryjournal-1">[1]</a></sup>
</p><p>Beyond PINES, the Evergreen ILS is deployed worldwide in approximately 1,800 libraries, and is used to power a number of statewide consortial catalogs.<sup class="reference" id="cite_ref-libraries_2-0"><a href="#cite_note-libraries-2">[2]</a></sup><sup class="reference" id="cite_ref-michigan_3-0"><a href="#cite_note-michigan-3">[3]</a></sup><sup class="reference" id="cite_ref-indiana_4-0"><a href="#cite_note-indiana-4">[4]</a></sup><sup class="reference" id="cite_ref-sitka_5-0"><a href="#cite_note-sitka-5">[5]</a></sup><sup class="reference" id="cite_ref-sclends_6-0"><a href="#cite_note-sclends-6">[6]</a></sup>
</p><p>In 2007,<sup class="reference" id="cite_ref-7"><a href="#cite_note-7">[7]</a></sup> the original Evergreen development team formed a commercial company around the software, <a class="new" href="/w/index.php?title=Equinox_Software&amp;action=edit&amp;redlink=1" title="Equinox Software (page does not exist)">Equinox Software</a>, which provides custom support, development, migration, training, and consultation for Evergreen.  As of 2014, several more companies and groups also provide support and related services for Evergreen.<sup class="reference" id="cite_ref-evergreen_companies_8-0"><a href="#cite_note-evergreen_companies-8">[8]</a></sup>
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Evergreen_(software)&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Evergreen was developed by the <a href="/wiki/Georgia_Public_Library_Service" title="Georgia Public Library Service">Georgia Public Library Service</a> (GPLS) to support 252 public libraries in the <a href="/wiki/Public_Information_Network_for_Electronic_Services" title="Public Information Network for Electronic Services">Public Information Network for Electronic Services</a> (PINES) consortium.<sup class="reference" id="cite_ref-9"><a href="#cite_note-9">[9]</a></sup>  Development began in June 2004 when state librarian Lamar Veatch announced in an open letter that after reviewing options available, GPLS decided to develop its own library automation system.  GPLS believed it could develop a system customized to fit its needs better at a lower cost than the fees currently being paid.<sup class="reference" id="cite_ref-10"><a href="#cite_note-10">[10]</a></sup>  Programmers in the GPLS developed the project for two years, and PINES successfully completed the transition to Evergreen in September 5, 2006.  In the next two years, the PINES consortium increased to over 270 libraries and five other systems in the United States and Canada implemented Evergreen.
</p><p>The software started receiving contributions from other libraries and developers in 2007.<sup class="reference" id="cite_ref-11"><a href="#cite_note-11">[11]</a></sup> 2009 saw the first Evergreen International Conference.<sup class="reference" id="cite_ref-12"><a href="#cite_note-12">[12]</a></sup> In 2012, the community joined the <a href="/wiki/Software_Freedom_Conservancy" title="Software Freedom Conservancy">Software Freedom Conservancy</a> and formed an oversight board.<sup class="reference" id="cite_ref-13"><a href="#cite_note-13">[13]</a></sup>
</p><p>Other Evergreen implementations in North America:<sup class="reference" id="cite_ref-14"><a href="#cite_note-14">[14]</a></sup>

<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Evergreen_(software)&amp;action=edit&amp;section=2" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Development priorities for Evergreen are that it be stable, robust, flexible, secure, and user-friendly.
</p><p>Evergreen's features include:
</p>
<ul><li>Circulation: for staff to check items in and out to patrons</li>
<li><a href="/wiki/Library_catalog" title="Library catalog">Cataloging</a>: to add items to the library’s collection and input information, classifying and indexing those items.  Evergreen is known for an extremely flexible indexing system that allows for a high level of customization and by default uses Library of Congress <a class="external text" href="https://www.loc.gov/standards/mods/" rel="nofollow">MODS</a> as its standard.</li>
<li><a href="/wiki/Online_public_access_catalog" title="Online public access catalog">Online public access catalog</a> (OPAC): a public catalog, or discovery interface, for patrons to find and request books, view their account information, and save book information in Evergreen "bookbags." The OPAC received a makeover in early 2009 with the new, optional skin, Craftsman.  There is also an optional Children's OPAC.  Various patron services such as paying bills by PayPal and Stripe, optional retaining of circulation history, book bags and more.</li>
<li>Self Service - Evergreen comes with self checkout and registration options that can be activated by the libraries.</li>
<li>OPAC exposes structured web data by schema.org standards to aid discovery by major search engines.</li>
<li>Acquisitions:  for staff to keep track of those materials purchased; invoices, purchase orders, selection lists, etc.</li>
<li>Authorities</li>
<li>Serials</li>
<li>Web based staff client that is OS independent</li>
<li>Added Content services Chillifresh, Content Cafe, Novelist, OpenLibrary and Syndetics natively supported with others supportable.</li>
<li>Native APIs for writing custom clients.</li>
<li>Statistical Reporting: flexible, powerful reporting for retrieval of any statistical information stored in the database.</li>
<li><a href="/wiki/Standard_Interchange_Protocol" title="Standard Interchange Protocol">SIP 2.0</a> support: for interaction with computer management software, self-check machines, and other applications.</li>
<li><a href="/wiki/Search/Retrieve_via_URL" title="Search/Retrieve via URL">Search/Retrieve via URL</a> and <a href="/wiki/Z39.50" title="Z39.50">Z39.50</a> servers</li></ul>
<p>Evergreen also features the Open Scalable Request Framework (OpenSRF, pronounced 'open surf'), a stateful, decentralized service architecture that allows developers to create applications for Evergreen with a minimum of knowledge of its structure.<sup class="reference" id="cite_ref-16"><a href="#cite_note-16">[16]</a></sup>
</p>
<h2><span class="mw-headline" id="Languages">Languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Evergreen_(software)&amp;action=edit&amp;section=3" title="Edit section: Languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The business logic of Evergreen is written primarily in <a href="/wiki/Perl" title="Perl">Perl</a> and <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a>, with a few optimized sections rewritten in C. The catalog interface is primarily constructed using <a href="/wiki/Template_Toolkit" title="Template Toolkit">Template Toolkit</a> with some <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>.  The staff client user interface is written in Mozilla's <a href="/wiki/XUL" title="XUL">XUL</a> (<a href="/wiki/XML" title="XML">XML</a> + JavaScript) before 3.0 and is a web based staff client built using AngularJS and related technologies as of 3.0.  <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> is used for the internationalization build infrastructure. <a class="mw-redirect" href="/wiki/Electronic_Data_Interchange" title="Electronic Data Interchange">EDI</a> functionality for the acquisitions system depends upon <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> support but work targeting 3.0 or later will remove this to handle EDI generation natively.<sup class="reference" id="cite_ref-17"><a href="#cite_note-17">[17]</a></sup>
</p>
<h2><span class="mw-headline" id="Requirements">Requirements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Evergreen_(software)&amp;action=edit&amp;section=4" title="Edit section: Requirements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Evergreen runs on <a href="/wiki/Linux" title="Linux">Linux</a> servers and uses <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a> for its backend database.  The staff client used in day-to-day operations by library staff runs on <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>, <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a>, or Linux computers and is built on <a href="/wiki/XULRunner" title="XULRunner">XULRunner</a>, a Mozilla-based runtime that uses the same technology stack as <a class="mw-redirect" href="/wiki/Mozilla_Firefox" title="Mozilla Firefox">Firefox</a> and allows for a browser-independent offline mode.  The online public access catalog (OPAC) used by library patrons is accessed in a <a href="/wiki/Web_browser" title="Web browser">web browser</a>.  As of version 3.0 due for release in late 2017 the technical preview of the web based staff client will be promoted to production use and the XUL based staff client that required local machine installation will begin being phased out.  <sup class="reference" id="cite_ref-18"><a href="#cite_note-18">[18]</a></sup>
</p>
